# YapaMarket - Astro SSR Multi-Domain Setup Guide

## 📋 Overview

This is a fully configured Astro SSR (Server-Side Rendering) application designed to run on AWS Amplify Hosting with multi-domain and multi-language support.

### Features

- ✅ **Server-Side Rendering (SSR)** - Dynamic content generation on the server
- ✅ **Multi-Domain Detection** - Separate language/content based on domain
- ✅ **Multi-Language Support** - English and Spanish based on domain
- ✅ **AWS Amplify Ready** - Complete deployment configuration
- ✅ **Status Endpoint** - `/status` page showing SSR status and Node.js version

## 🚀 Quick Start

### Prerequisites

- Node.js 18+ 
- npm or yarn

### Local Development

1. **Install dependencies**
   ```bash
   npm install
   ```

2. **Start development server**
   ```bash
   npm run dev
   ```
   
   The server will start on `http://localhost:3000`

3. **Build for production**
   ```bash
   npm run build
   ```

4. **Preview production build**
   ```bash
   npm run preview
   ```

## 🌐 Domain Configuration

The application automatically detects the domain and switches language:

| Domain | Language | Title | Path |
|--------|----------|-------|------|
| `https://new.yapamarket.com/` | English | YapaMarket / EN | `/` |
| `https://nuevo.yapamarket.com/` | Spanish | YapaMarket / ES | `/` |

Both domains also have a `/status` endpoint that:
- Shows the active Node.js version (`process.version`)
- Confirms SSR is running
- Displays server information and timestamp
- Shows detected language

## ⚙️ Configuration Files

### astro.config.mjs
```javascript
export default defineConfig({
  output: 'server',              // Enable SSR
  adapter: node({
    mode: 'standalone',          // Standalone Node.js server
  }),
  server: {
    host: '0.0.0.0',             // Listen on all interfaces (required for Amplify)
    port: 3000,
  },
});
```

### amplify.yml
- **Install Phase**: Installs npm dependencies (`npm ci`)
- **Build Phase**: Builds the Astro project (`npm run build`)
- **Artifacts**: Serves the `dist/` directory
- **Caching**: Caches `node_modules` for faster builds

## 🌍 How Domain Detection Works

The `src/pages/index.astro` and `src/pages/status.astro` files use server-side logic to detect the domain:

```javascript
const host = Astro.url.host;

if (host.includes('nuevo.yapamarket.com')) {
  language = 'ES';
} else if (host.includes('new.yapamarket.com')) {
  language = 'EN';
}
```

This is **server-side rendering**, so the detection happens on the server and content is generated dynamically for each request.

## 📡 AWS Amplify Deployment

### Prerequisites on AWS Amplify

1. Connect your GitHub/GitLab repository
2. Ensure Node.js 18+ is available in the build environment
3. Point your domains to the Amplify hosting

### Build Configuration

The `amplify.yml` file is automatically read by Amplify and contains:

- **Frontend build commands** - `npm ci` and `npm run build`
- **Artifact directory** - `dist/` (generated by Astro build)
- **Caching** - Node modules are cached between builds

### Deployment Steps

1. Push code to your repository
2. AWS Amplify automatically detects `amplify.yml`
3. Amplify runs the build phases and deploys to CDN
4. Your SSR server serves dynamic content from the Amplify runtime

## 🛠️ Customization

### Adding New Pages

Create new `.astro` files in `src/pages/`:

```astro
---
// src/pages/about.astro
const host = Astro.url.host;
const isSpanish = host.includes('nuevo.yapamarket.com');
---

<html>
  <head><title>{isSpanish ? 'Acerca de' : 'About'}</title></head>
  <body>
    <h1>{isSpanish ? 'Acerca de' : 'About'}</h1>
  </body>
</html>
```

### Adding More Domains

Edit `src/pages/index.astro` and `src/pages/status.astro`:

```javascript
if (host.includes('nuevo.yapamarket.com')) {
  language = 'ES';
} else if (host.includes('new.yapamarket.com')) {
  language = 'EN';
} else if (host.includes('autre.yapamarket.com')) {
  language = 'FR';  // Add French support
}
```

### Environment Variables

Create a `.env` file for environment-specific settings:

```env
NODE_ENV=production
```

Access in pages using `import.meta.env.NODE_ENV`

## 🚨 Troubleshooting

### Build fails on Amplify

Check that:
1. `astro.config.mjs` has `output: 'server'`
2. `@astrojs/node` adapter is installed
3. `amplify.yml` has correct build commands
4. Node.js version is 18+

### Domain detection not working

Verify:
1. Domains are properly routed to Amplify hosting
2. Both domains point to the same Amplify app
3. Domain names in code match your actual domains

### SSR not active

The `/status` page shows server timestamp. If it's always the same or format looks wrong:
1. Ensure `output: 'server'` in `astro.config.mjs`
2. Not using static export (which would be `output: 'static'`)
3. Amplify is serving the Node.js server, not just static files
